## 困ったときの逆引き

問題文の特徴 → スニペット

### 制約サイズで分類
- **N ≤ 20**: bit DP, `perm` (全順列), `bitsubset` (部分集合 DP)
- **N ≤ 5000**: O(N²) DP, `bimatch`, `lis`
- **N ≤ 2·10⁵**: セグ木系 (`bit` / `segtree` / `lazysegtree`), `sqrtdecomposition`, `mos`, `LCA`, `hld`
- **N ≤ 10⁶**: O(N) または O(N log N) のみ。`bit`, `zalgorithm`, `suffixautomaton`, `slidemin`
- **値域 10¹⁸**: 座圧して `segtree`、または `dynamicsegtree`
- **64bit 越え**: `int128`、桁数 100+ なら `bigint`

### 区間クエリの組合せ
- 区間和取得 + 点更新 → `bit`
- 区間モノイド (min/max/gcd) + 点更新 → `segtree`
- **区間更新 + 区間クエリ** → `lazysegtree` / `lazysegtreev2`
- 区間更新 + 点取得 → `dualsegtree`
- 区間 chmin/chmax/add/sum 混在 → `segtreebeats`
- 区間 k 番目 / 値域カウント → `mergesorttree` または `wavelet matrix` (スニペット無)
- オフライン区間クエリ Q 個 → `mos`

### 木の問題
- 2 頂点間距離 → `LCA` (距離 = `dep[u]+dep[v]-2*dep[lca]`)
- 部分木の合計/最大 → `eulertour` + `bit` or `segtree`
- パス u-v への加算 + パス和 → `hld` + `lazysegtree`
- 全頂点を根にした答え → `rerooting`
- 直径・離心率 → `treeeccentricity`
- 動的な辺追加/削除 + パスクエリ → `linkcuttree`
- 「距離 d 以下のペアを数える」系 → `treecentroid`
- 指定頂点集合のみ残した木 DP → `auxiliarytree`

### 文字列の問題
- 周期検出、`S[i..]` と `S` の一致長 → `zalgorithm`
- 異なる部分文字列の個数 → `suffixautomaton` (または `suffixarray` で `ΣLCP`)
- 辞書順 K 番目の部分文字列 → `suffixarray`
- 複数パターンを同時検索 → `ahocorasick`
- 集合中で v との XOR 最大 → `prefixtree` (二進 Trie)

### グラフの問題
- 順序付け・依存関係 → `tsort`
- 「サイクルを 1 個に潰したい」 → `scc`
- 辺/頂点を消すと連結が増えるか → `lowlink`
- 最大流 / 最小カット → `dinic` (汎用) / `maxflow` (小規模)
- 二部マッチング → `bimatch` (小) / `hopcroftkarp` (大)
- 割当・輸送・重み付きマッチング → `mincostmaxflow`
- 差分制約 (`x_j - x_i ≤ c`) → `cowgame`
- 最小全域木 → `kruskal`

### DP 系
- `dp[i] = min_j(a_j · x_i + b_j) + c_i` → `cht`
- 分区線形凸関数の min を維持 → `slopetrick`
- 単峰関数の極値 → `trisearch`
- ヒストグラム/2D 0/1 行列の最大長方形 → `maxrectangle`
- 線形漸化式の N 項目 → `matrix` 累乗

### ゲーム/その他
- 公平ゲームの先手必勝判定 → `grundy`
- 集合 XOR の構造 (最大値, 到達可能性) → `xorbasis`
- 順列を恒等に戻す最小 swap, k 回後の状態 → `cycledecomposition`
- k 回後の遷移先 (functional graph) → `doubling` / `doublingv2`
- 環状の距離 → `ring`
- 4/8 方向走査 → `didj`
- 順位/k 番目クエリ → `orderedset` (簡単) / `splaytree` (拡張)
- 過去の状態を参照 → `persistentunionfind`, `rollbacksegtree`

## メモ (trigger key の罠)

- `trinaly_search` の trigger は **`trisearch`** (短い方)。
- `int128` の trigger は **`gnu128bitint`**。
- `lca` の trigger は **大文字 `LCA`**。
- `mos` (Mo's algorithm) の trigger は `mos` (短縮)。
- `splaytree` (ファイル名 `splay`)、`linkcuttree` (`link_cut_tree`)、`treecentroid` (`tree_centroid_decomposition`)、`treeeccentricity` (`tree_eccentricity`) — ファイル名と trigger が異なるシリーズ。
- `atcoderlazysegtree` (ファイル名 `atlazyseg`) — ファイル名が短く trigger が長い唯一の逆転。
- `cht` (`convex_hull_trick`)、`slopetrick` (`slope_trick`)、`maxrectangle` (`max_rectangle`) — アンダースコア無し型。
